# 🚀 Streamlit Application

This folder contains the **Streamlit web application** for the **Telecom Customer Churn Prediction** project.

The application provides an interactive web interface that allows users to enter customer information and instantly predict whether a customer is likely to churn using a trained Machine Learning model.

---

# ✨ Features

- 📊 Real time customer churn prediction
- 🤖 Random Forest based Machine Learning model
- 🎯 Interactive and user friendly interface
- ⚡ Instant prediction results
- 📈 Business friendly analytics dashboard
- 📉 Model performance evaluation
- ⭐ Feature importance visualization

# ▶️ How to Run the Project

### 1. Clone the repository

```bash
git clone https://github.com/alanbinu/Telecom-Customer-Churn-Prediction.git
```

### 2. Navigate to the project directory

```bash
cd Telecom-Customer-Churn-Prediction
```

### 3. Install the required dependencies

```bash
pip install -r requirements.txt
```

### 4. Launch the Streamlit application

Since the Streamlit application is located inside the **app** folder, run:

```bash
streamlit run app/app.py
```

### 5. Open the application

If the browser does not open automatically, visit:

```
http://localhost:8501
```

---

# 🧠 Application Workflow

1. The user enters customer information through the Streamlit interface.
2. The input data is preprocessed using the trained `StandardScaler`.
3. The data is transformed to match the feature format used during model training.
4. The trained Random Forest classifier predicts whether the customer is likely to churn.
5. The prediction probability and business recommendations are displayed on the dashboard.

# ❗ Troubleshooting

### `Error: File does not exist: app.py`

This occurs if you run:

```bash
streamlit run app.py
```

Instead, run:

```bash
streamlit run app/app.py
```

because the Streamlit application is located inside the **app** folder.

---

### Model file not found

If you receive an error indicating that the model or scaler cannot be found, verify that the following files exist inside the **models** folder:

```
models/
├── telecom_churn_Project.pkl
└── Scaler.pkl
```

---

# 📄 Model Files

The trained Machine Learning model files (`telecom_churn_Project.pkl` and `Scaler.pkl`) may not be included in this repository if they exceed GitHub's file size limits.

If the model files are unavailable, they can be regenerated by running the notebook provided in the **notebooks/** directory and saving the trained model and scaler into the **models/** folder.
